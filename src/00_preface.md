# はじめに {.unnumbered}

## 本書の背景と目的

### 執筆のきっかけ

技術書や専門書の執筆において、従来のワークフローには多くの課題がありました。WordやGoogle Docsでの執筆では、複数の執筆者との共同作業が困難で、バージョン管理も煩雑です。また、EPUBへの変換作業は手動で行うことが多く、品質の一貫性を保つのが困難でした。

2025年現在、KindleはEPUBネイティブ対応となり、電子書籍フォーマットは EPUB に統一されつつあります。このタイミングで、**Markdown + Pandoc** を用いた効率的な電子書籍制作ワークフローを確立することは、多くの著者にとって重要な課題となっています。

### 本書の目的

本書では、以下の目標を達成することを目指しています：

1. **効率的な執筆環境の構築**：Markdown による構造化された原稿作成、Git を活用したバージョン管理、エディタの支援機能を最大限活用

2. **自動化されたビルドプロセス**：1コマンドでのEPUB生成、品質チェックの自動化、プレビューの即座確認

3. **継続的インテグレーション（CI/CD）**：GitHub Actions による自動ビルド、品質ゲートの設定、自動リリース

4. **保守性の高いワークフロー**：再利用可能なテンプレート、拡張しやすい設定、チーム開発への対応

### 対象読者とシステム要件

本書は **macOS Sequoia 15.5** 以降で Homebrew を利用できるエンジニア・クリエイターを主な対象としています。ただし、基本的な概念は他のオペレーティングシステムでも応用可能です。

## 必要な前提知識

### コマンドライン操作

本書では、ターミナルアプリケーションを使用したコマンドライン操作を多用します。以下の操作に慣れておくことを推奨します：

```bash
# ディレクトリ操作
cd /path/to/directory
ls -la
mkdir new-directory
pwd

# ファイル操作
cp source.txt destination.txt
mv oldname.txt newname.txt
rm unwanted-file.txt
find . -name "*.md"

# プロセス管理
ps aux | grep pandoc
kill -9 PID
jobs
```

### Git によるバージョン管理

本書で構築するワークフローでは、Git を使用したバージョン管理が前提となります：

```bash
# 基本的な Git 操作
git init
git add .
git commit -m "Update content"
git push origin main

# ブランチ操作
git branch feature/new-chapter
git checkout feature/new-chapter
git merge feature/new-chapter

# リモートリポジトリ操作
git remote add origin https://github.com/user/repo.git
git pull origin main
```

## 本書の構成と学習方法

### 章立ての設計思想

本書は、**段階的な学習アプローチ**を採用しています：

1. **概念理解**（第1章）：全体像の把握、技術選択の理由、期待される成果

2. **基礎構築**（第2-3章）：必要なツールの習得、環境構築、基本操作の習得

3. **実践応用**（第4-5章）：実際のワークフロー構築、高度な機能の活用、カスタマイズ手法

4. **運用・保守**（第6-7章）：自動化の実現、問題解決、継続的改善

### 推奨する学習方法

**ハンズオン中心のアプローチ**

各章には実践的な演習問題を配置しています。理論の学習と並行して、実際に手を動かすことを強く推奨します：

1. **環境構築（所要時間：2-3時間）**：まず第3章で環境構築を完了、基本的な変換動作を確認

2. **基本ワークフローの習得（所要時間：4-6時間）**：第4章でシンプルなプロジェクトを作成、自動ビルドの動作を確認

3. **段階的な機能拡張（所要時間：6-8時間）**：第5章で高度な機能を追加、自分のニーズに合わせてカスタマイズ

4. **本格運用（所要時間：4-6時間）**：第6章で CI/CD を設定、実際の書籍プロジェクトで活用

**学習進度の目安**

- **初心者**：2-3週間で基本ワークフローを習得
- **中級者**：1-2週間で高度なカスタマイズまで完了
- **上級者**：数日で全機能を把握し、独自拡張を開始
